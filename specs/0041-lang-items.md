## 0041: Lang Items — the `@lang` Attribute

Status: Draft

コンパイラが役割で認識する必要のある宣言を，名前ではなく属性 `@lang("<役割名>")` で束縛する（2026-07-19 決定）．

いくつかの型・関数は，stdlib のソースで定義されつつも，コンパイラがその**役割**を知っていなければならない（例：`Option`（0042）の variant をベアで構築・照合し match の網羅性を検査するには，コンパイラが「どの enum が Option か」を知る必要がある）．本仕様は，こうした宣言をコンパイラの既知の役割へ束縛する汎用機構 **lang-item** を定める．束縛は属性 `@lang("<役割名>")`（0039）で行い，コンパイラが保持するのは「役割 → 宣言」の対応のみである．宣言の実体（enum なら variant の定義など）は stdlib ソース側にある．役割名の集合は言語仕様が閉じて定め，各役割の対象宣言種・形状要件・特別扱いの内容は，その役割を導入する仕様が定める．本仕様時点で認識される役割は `option`（0042）のみである．

### Summary

- **lang-item** は，コンパイラが役割で認識する宣言である．属性 `@lang("<役割名>")`（0039）で宣言を役割へ束縛する．
- 役割名の集合は言語仕様が閉じて定める．認識されない役割名はコンパイルエラーとする．各役割の意味・対象宣言種・形状要件は，それを導入する仕様が定める（本仕様は枠のみを定める）．
- 1 つの役割は高々 1 つの宣言に束縛される．同一役割の二重束縛はエラーとする．束縛された宣言が役割の形状要件に反する場合もエラーとする．
- コンパイラは，束縛された宣言に対してのみ役割の特別扱いを与える．役割を宣言の**名前やモジュールで識別してはならない**（名前脱結合）．
- 本仕様時点の役割は `option`（0042）のみである．

### Motivation

1. **名前ではなく役割で束縛する**．コンパイラが特定の型を名前（`Option`）で特別扱いすると，名前・モジュールへの結合がフロントエンド各所（パーサ・型検査・脱糖・LSP）に散る．`@lang` で束縛すれば，コンパイラと stdlib の契約は定義箇所の 1 行に集約され，定義を読めば役割を持つことが分かる（自己文書化）．これは 0039 が属性を「宣言に前置するメタデータの一点」に絞ったのと同じ設計の延長である．
2. **核を小さく（0000）**．0021 が算術・比較の意味を stdlib の intrinsic へ，0038 が prelude を埋め込みソースへ出したのと同じ方向で，特別扱いされる型の**定義**を stdlib へ出し，コンパイラには「役割の束縛」だけを残す．
3. **再利用可能なレジストリ**．最初の需要は `Option` の非組み込み化（0042）だが，同じ機構は将来 `Result` 等を stdlib の値型へ降ろす際にも `@lang("result")` として再利用できる（役割を 1 つ足すだけ）．役割ごとにコンパイラ内へ名前照合を書き足す方式より拡張が局所化する．

### Specification

以下，キーワードは RFC 2119 に従う．

#### lang-item 束縛（L 規則）

- **L1（束縛と役割レジストリ）**: 属性 `@lang("<役割名>")`（0039）は，前置された宣言をコンパイラの既知の役割へ束縛する．役割名は言語仕様が閉じて定める集合の要素でなければならない（MUST）．認識されない役割名はコンパイルエラーである（MUST）；診断は認識される役割の一覧を示すべきである（SHOULD）．各役割は，それを導入する仕様が，対象となる宣言種・形状要件・コンパイラが与える特別扱いを規範として定める（MUST）．本仕様時点の役割は次のとおりである：

  | 役割名 | 導入仕様 | 対象宣言種 |
  |---|---|---|
  | `option` | 0042 | `enum` |

- **L2（一意性と形状）**: 各役割は高々 1 つの宣言に束縛される（MUST）．同一役割名を複数の宣言に付けた場合はエラーである（MUST）．役割の対象宣言種（L1 表）と異なる宣言種に付けた場合（0039 R6），および役割の形状要件（導入仕様が定める）に反する場合はエラーである（MUST）．
- **L3（役割の特別扱いと名前脱結合）**: コンパイラは，役割に束縛された宣言に対してのみ，当該役割に定められた特別扱い（構築・照合・脱糖など）を与える（MUST）．コンパイラは役割を宣言の名前・モジュールで識別してはならない（MUST NOT）：束縛されていない同名・同形の宣言には，いかなる特別扱いも与えない．役割の解決は，束縛された宣言から役割固有の情報（例：variant の名前・タグ・arity）を読み取って行う．

### Examples

enum を `option` 役割へ束縛する（意味は 0042 が定める）：

```emela
@lang("option")
enum Option<T> {
    Some(T)
    None
}
```

認識されない役割名はエラー（L1）：

```emela
@lang("maybe")
enum Maybe<T> { Just(T), Nothing }
-- error: unknown lang-item role `"maybe"`; recognized roles: `"option"` (spec 0042)
```

同一役割の二重束縛はエラー（L2）：

```emela
@lang("option")
enum Option<T> { Some(T), None }

@lang("option")
enum Maybe<T> { Just(T), Nothing }
-- error: lang-item role `"option"` is already bound to `Option`
```

### Compilation Notes

この節は非規範的な実装上の補足である．

- lang-item テーブルは，宣言の登録時（enum なら enum 登録時）に構築する：`@lang("<役割>")` を持つ宣言を検出し，役割ごとの情報（enum なら variant の名前・タグ・arity）を記録して形状を検証する．以降のフロントエンド（構築・照合・脱糖・LSP）はこのテーブルを参照し，宣言の名前を直接照合しない．
- 役割名は 0039 の属性引数（文字列リテラル）として渡る．引数のパースと保持は 0039 の Compilation Notes に従う．
- 未束縛（役割に対応する `@lang` が無い）の状態でコンパイラがその役割を必要とする場合（例：embedded core が Option を提供しない）の扱いは，各役割の導入仕様（および 0038 の embedded core 供給）が定める．

### 他仕様との関係

- **0039（Attributes）**: `@lang` を引数付き認識属性として用いる（R4 表・R7 引数形）．属性の形（前置・重複・引数）は 0039 が，役割束縛の意味は本仕様（L 規則）が定める．`@test`（0040）と同じく「属性の枠は 0039，意味は導入仕様」という分担である．
- **0042（Option as a Core-Prelude Enum）**: `option` 役割を最初の役割として導入する．役割の形状（型引数 1・present/absent variant）と特別扱い（ベア構築・照合，match）は 0042 が定める．
- **0038（Core Package Embedding）**: 役割に束縛される宣言は，多くが embedded core / stdlib のソースに置かれる（`option` は std.core）．
- **0000（Language Design Principles）**: 特別扱いされる型の定義を stdlib へ出し，コンパイラに役割束縛のみを残すのは「核を小さく」の具体化である．

### Open Questions

- 役割名の集合をどこまで広げるか（`result` / `list` 等の値型を将来 stdlib へ降ろす需要次第）．役割は本仕様の枠の上に導入仕様を足す形で増やす．
- enum 以外の宣言種（関数・trait 等）への役割束縛の需要と，その形状要件の書き方．
- variant 単位の役割束縛（`@lang("some")` 等）．variant への属性付与は 0039 R2 の対象外（非トップレベル）であり，別途拡張を要する．0042 は variant の役割を arity で識別することでこれを回避している．
- 役割が要求され，かつ束縛が無い場合の診断（embedded core の破損検出）を本仕様で規定するか，各導入仕様に委ねるか．
