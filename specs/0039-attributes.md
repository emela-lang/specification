## 0039: Attributes

Status: Draft

属性トークンは `@` とし，属性名の集合は閉じて定める（2026-07-18 決定）．引数形 `@ident(arg)` は引数を取る属性（`@lang`，0041）のために解禁する（2026-07-19 決定，R7 の将来拡張条項に基づく）．

トップレベル宣言に前置するメタデータ構文 **attribute**（`@ident`）を導入する仕様．認識される属性名の集合は言語仕様が閉じて定め，各属性の意味と適用可能な宣言種はそれを導入する仕様が定める．本仕様時点で認識される属性は `test`（0040）と `lang`（0041）である．未知の属性はコンパイルエラーとする．引数形 `@ident(arg)` は引数を取る属性のために用い，引数の一般文法は各属性の導入仕様が規範として定める（本仕様は最小の枠のみを定める）．

### Summary

- `@ident` はトップレベルのアイテム宣言（`fn` / `enum` / `trait` / `impl` / `effect` / `extern fn`）に前置できるメタデータである．`module` ヘッダと `import` には付けられない．
- 認識される属性名は言語仕様が定める閉集合である．未知の属性は警告ではなくコンパイルエラーとする．
- 1 つの宣言に複数の属性を付けてよいが，同名の重複はエラーとする．
- `@ident(arg)` の引数形は引数を取る属性のために用いる．引数を取らない属性に `(...)` を付けるとエラーとする．引数の一般文法は各属性の導入仕様が定める（本仕様時点で引数を取る属性は `@lang`（0041）で，単一の文字列リテラルを取る）．
- 正規形（0035）は属性 1 個につき 1 行，宣言と同一インデント，宣言との間に空行を置かない．

### Motivation

1. **キーワードを増やさない宣言マーカー**．最初の需要は単体テスト宣言 `@test`（0040）である．`test fn` のような修飾子キーワードは宣言文法を宣言種ごとに広げるが，attribute は「宣言に前置するメタデータ」という一点だけを文法に足し，将来の deprecation 表明や lint 制御（0035 Open Question の抑制コメントはこの機構で解決しうる）にも同じ形で使える．
2. **未知はエラー（閉集合）**．属性は宣言の扱い（テストか否か，成果物に入るか否か）を変える．typo（`@tset`）を黙って無視すると，テストが静かに実行されなくなる．認識できない属性は必ずエラーにする．
3. **トークンは `@`**．現行文法で未使用であり（lexer に `@` は存在しない），1 トークンで完結して `#[...]` より軽い．

### Specification

以下，キーワードは RFC 2119 に従う．

#### 構文

- **R1（形）**: `attribute := "@" ident`．`@` と識別子の間に空白を置いてはならない（MUST）．属性は修飾対象の宣言に前置する．
- **R2（位置）**: 属性はトップレベルのアイテム宣言（`fn` / `enum` / `trait` / `impl` / `effect` / `extern fn` / `intrinsic fn`）に前置できる（MAY）．`pub` を持つ宣言では，属性は `pub` より前に置く（MUST）．`module` ヘッダ・`import` に付けてはならない（MUST NOT）．トップレベル以外の位置（trait / impl のメソッド，effect ブロック内の操作，式，仮引数）に置いてはならない（MUST NOT）．属性と宣言の間にコメント行が挟まってよい（コメントは trivia であり，属性と宣言の結び付きを変えない）．
- **R3（複数・重複）**: 1 つの宣言に複数の属性を前置してよい（MAY）．同名の属性の重複はエラーとする（MUST）．

#### 意味論

- **R4（閉集合）**: 認識される属性名の集合は言語仕様が閉じて定める．各属性は，それを導入する仕様が意味と適用可能な宣言種を定める．本仕様時点の集合は次のとおりである：

  | 属性 | 導入仕様 | 適用対象 | 引数 |
  |---|---|---|---|
  | `test` | 0040 | トップレベル `fn` | なし |
  | `lang` | 0041 | トップレベル `enum` | 文字列リテラル 1 個（lang-item の役割名） |

- **R5（未知属性）**: 未知の属性名はコンパイルエラーである（MUST）．警告にしてはならない（MUST NOT）．診断は認識される属性の一覧を示すべきである（SHOULD）．
- **R6（適用性）**: 認識される属性を，その属性が適用できない宣言種に付けた場合はエラーとする（MUST）．
- **R7（引数）**: 属性は括弧付き引数 `@ident(arg)` を取れる（MAY）．引数を取る認識属性は，その導入仕様が引数の形（個数・種別）を規範として定める（MUST）．引数を取らない属性に `(...)` を付けた場合，および認識属性の要求する引数形に合致しない場合はエラーとする（MUST）．本仕様時点で引数を取る認識属性は `@lang`（0041）であり，単一の文字列リテラルを取る．引数の一般文法（複数引数・名前付き引数・リテラルの種別）は本仕様では定めず，将来仕様で本仕様を supersede せず拡張できる（Open Question 参照）．

#### 正規整形（0035 拡張）

- **R8（正規形）**: 属性の正規形（0035 F 規則の拡張）は次のとおりである（MUST）：属性 1 個につき 1 行，修飾対象の宣言と同一インデント，`@` は属性名に密着，属性と宣言（または次の属性）の間の空行は除去する．属性と宣言が同一行に書かれた場合は属性を独立行へ分離する．属性の並べ替えは行わない（F9 と同旨）．

### Examples

正当な形（`@test` の意味は 0040 が定める）：

```emela
@test
fn append_keeps_order() -> Unit uses {} {
    ...
}
```

未知の属性はエラー：

```emela
@deprecated
fn old_api() -> Unit uses {} { () }
-- error: unknown attribute `@deprecated`; recognized attributes: `@test` (spec 0040)
```

同名の重複はエラー：

```emela
@test
@test
fn twice() -> Unit uses {} { () }
-- error: duplicate attribute `@test`
```

引数を取る属性（`@lang`，意味は 0041 が定める）：

```emela
@lang("option")
enum Option<T> {
    Some(T)
    None
}
```

引数を取らない属性に引数を付けるとエラー（R7）：

```emela
@test("append keeps order")
fn append_keeps_order() -> Unit uses {} { () }
-- error: attribute `@test` takes no arguments (spec 0040)
```

認識されない役割名を `@lang` に渡すとエラー（役割名の集合は 0041 が閉じて定める）：

```emela
@lang("maybe")
enum Maybe<T> { Just(T), Nothing }
-- error: unknown lang-item role `"maybe"`; recognized roles: `"option"` (spec 0042)
```

適用できない宣言種はエラー（R6，適用対象は 0040 が定める）：

```emela
@test
enum Color {
    Red
}
-- error: attribute `@test` may only be applied to a top-level `fn` (spec 0040)
```

整形（R8）——空行の除去と独立行への分離：

```emela
-- 整形前
@test

fn spaced() -> Unit uses {} { () }

-- 整形後
@test
fn spaced() -> Unit uses {} { () }
```

### Compilation Notes

この節は非規範的な実装上の補足である．

- lexer に `@` トークンを追加する．属性の解析は，トップレベルのアイテムディスパッチの前に属性列を収集し，直後のアイテムの AST に保持する形で実装できる．
- 未知属性（R5）・重複（R3）・適用性（R6）の検査は parse 後の検証パスで行い，0033 のエラー収集方針（1 件で停止せずすべて報告する）に乗せる．
- フォーマッタは属性をアイテムの付属として扱えば，0035 の F 規則への変更は R8 の追加だけで済む．属性はアイテムと一体で移動・整形され，単独では扱わない．
- 引数付き属性（`@lang`）は，`@ident` の直後の `(` を見て文字列リテラルを 1 個読み，属性ノードに引数として保持する．引数を取らない属性に `(...)` が付いた場合・要求する引数形に合致しない場合は R7 のエラーにする．属性値の消費（enum への lang-item 束縛）は各導入仕様の Compilation Notes が述べる（`@lang` は 0041）．

### 他仕様との関係

- **0035（Canonical Formatting and Lint）**: R8 が F 規則を拡張する．lint の L 規則は不変であり，属性起因の新しい lint は導入しない．Open Question の警告抑制コメント（`-- lint:allow(rule-id)`）は，将来 R7 の引数文法が定まれば属性（`@allow(...)` 等）で置き換えうる．
- **0033（Language Server）**: 属性の診断（R3 / R5 / R6）はエラー収集（複数報告）の対象である．属性名の補完は将来課題．
- **0034（Multiline Expressions）**: 属性は常に独立行であり（R8），継続行規則の影響を受けない．
- **0003 ほか宣言系仕様**: 宣言文法に属性前置の形が加わる．意味は各属性の導入仕様が定める（R4）．
- **0040（Unit Testing）**: `test` を最初の認識属性として登録する．
- **0041（Lang Items）**: `lang` を最初の引数付き認識属性として登録する（`@lang("<役割名>")`）．`@lang` は宣言を lang-item（コンパイラが役割で認識する宣言）へ束縛する汎用機構であり，R7 の引数形を初めて実体化する．最初の役割 `option` は 0042 が定める．

### Open Questions

- 属性引数の一般文法（複数引数・名前付き引数・リテラルの種別）．R7 は最小の枠のみを定め，`@lang` の「文字列リテラル 1 個」を最初の実体とした．次の需要は `@test(expect_throw)`（0040 Open Question，識別子引数）と `@deprecated("...")` になる見込みで，これらを収容する一般文法をどこで定めるか（本仕様の追補か新仕様か）は未決．
- 非トップレベル位置（trait / impl メソッド，effect ブロック内の操作）への拡張．
- 名前空間付き属性（`@lint.allow(rule)` — 0035 の抑制コメント Open Question との接続）．
- ドキュメンテーション属性（doc comment との関係）．
- `module` / `import` への属性．
