## 0014: Generic Functions

Status: Draft

関数に型パラメータを持たせる **ジェネリック関数**（パラメトリック多相）を定義する仕様．

### Summary

- 名前付き関数は `fn name<T, U, ...>(...)` のように型パラメータを宣言できる．
- 型パラメータは引数型・戻り値型・`throws` 型・関数本体で型として使える．
- 型パラメータは **不透明 (opaque)** である．その値に対して具体型を要求する操作（算術，比較，`match`，フィールドアクセス）はできず，束縛・受け渡し・戻り・配列要素化・（関数型なら）適用のみができる．
- 呼び出し時の型引数は **実引数の型から推論** する．明示的な型引数構文は本仕様では導入しない．
- 制約 (bounds / trait) は持たない．完全なパラメトリック多相である．

### Motivation

`Array<T>` や `Option<T>` のような組み込みジェネリック型は既に存在するが，ユーザーは型に依存しない関数（恒等関数，`map`，`first` など）を一度しか書けず，型ごとに重複定義する必要があった．

ジェネリック関数の構文は既にいくつかの仕様で前方参照されている．

- 0008 は高階関数の例として `fn map<T, U>(xs: Array<T>, f: (T) -> U uses 'e) -> Array<U> uses 'e` を示す．
- 0011 は `fn head<T>(xs: Array<T>) -> T` を `panic` の例で用いる．
- 0003 / 0008 は effect row に対するジェネリクス（`fn apply<T>(..., uses 'e)` / `fn map<T, U>(..., uses 'e)`）を予告する（effect-row 多相は spec 0022 で定義する）．

本仕様はこのうち **型パラメータ** を正式に定義する．effect row に対するジェネリクス（effect-row 多相）は spec 0022 (Effect-Row Polymorphism) で定義する．error 型に対する行多相（union error）は将来仕様に送る (Open Questions)．

### Specification

#### 型パラメータの宣言

名前付き関数は，関数名の直後の `< >` に型パラメータを列挙して宣言する．

```emela
fn identity<T>(x: T) -> T uses {} {
    x
}
```

- 型パラメータは 1 つ以上をカンマ区切りで並べる: `<T>`, `<T, U>`．
- 型パラメータ名は型の位置に書ける識別子であり，慣習として大文字で始める．
- `< >`（空のリスト）は書けない．型パラメータを持たない関数は `< >` を省略する（従来通り）．

宣言された型パラメータは，そのシグネチャ（引数型・戻り値型・`throws` 型）と関数本体の中で **型** として使える．スコープはその関数定義に閉じる．異なる関数の同名型パラメータは無関係である．

```emela
fn map<T, U>(xs: Array<T>, f: (T) -> U uses {}) -> Array<U> uses {} {
    ...
}
```

ここで `T`, `U` は `map` の本体とシグネチャでのみ意味を持つ．

#### 不透明性 (parametricity)

型パラメータ `T` の具体型はコンパイル時に呼び出しごとに定まるが，ジェネリック関数の本体は `T` がどの具体型かを知らない．したがって `T` 型の値に対してできるのは次に限られる．

- `let` で束縛する
- 関数の引数として渡す
- 関数から返す
- 配列 `Array<T>` の要素にする / 取り出す
- `Option<T>` などジェネリックな組み込み型に入れる / 取り出す
- `T` が関数型のパラメータ（例 `f: (A) -> T`）なら適用する

`T` 型の値に対して具体型を要求する操作はコンパイルエラーである．

```emela
fn bad<T>(x: T) -> T uses {} {
    x + x          -- error: `+` requires Int or Float, not the opaque `T`
}
```

- 算術 `+ - *`，比較 `== <` はオペランドの具体的な数値型を要求するため `T` には適用できない．
- `match` は具体的な enum / `Option` の構造を要求するため `T` には適用できない．
- フィールドアクセスは具体的な record を要求するため `T` には適用できない．

この制約により，ジェネリック関数の観測可能な振る舞いは型引数によらず同一になる（parametricity）．

#### 型引数の推論

ジェネリック関数の呼び出しでは，型引数を **実引数の型から推論** する．明示的な型引数構文（例 `identity<Int>(x)`）は本仕様では導入しない（Open Questions）．

推論は，宣言上のパラメータ型（型変数を含む）と，実引数の具体型を構造的に対応付けて代入 `{T ↦ Int, ...}` を求める．

```emela
fn first<T>(xs: Array<T>) -> Option<T> uses {} {
    Array.get(xs, 0)
}

let xs: Array<Int> = [1, 2, 3]
let head = first(xs)      -- Array<T> と Array<Int> を対応付けて T = Int．結果は Option<Int>
```

対応付けは次の構造を貫く．

- 型変数 `T` ↔ 任意の具体型 `C` : `T = C` を記録する．既に別の具体型が記録されていれば矛盾でありエラー．
- `Array<D>` ↔ `Array<A>` : `D` と `A` を再帰的に対応付ける．
- `Option<D>` ↔ `Option<A>` : 同上．
- `(D1, ...) -> Dr` ↔ `(A1, ...) -> Ar` : 各引数型と戻り値型を再帰的に対応付ける．
- それ以外 : 具体型同士が一致していなければエラー．

すべての型パラメータが推論で一意に定まらなければならない．そのため，各型パラメータは **少なくとも 1 つのパラメータ型の中（ネストを含む）に出現していなければならない** (MUST)．いずれのパラメータ型にも現れない型パラメータを宣言する関数は，呼び出しから型引数を決定できないためコンパイルエラーとする．

```emela
fn pick<T>() -> T uses {}      -- error: `T` does not appear in any parameter type
```

`map<T, U>` の `U` は，引数 `f: (T) -> U` の戻り値位置に現れるため，渡された関数値の型から決定できる．

戻り値型・`throws` 型は，求めた代入を適用して具体化する．呼び出し式全体の型はこの具体化された戻り値型である．

#### effect と throws

ジェネリック関数も，具体的な effect row (`uses`) と error 型 (`throws`) を宣言できる（省略は従来通り）．**effect row を型変数にする多相（effect-row polymorphism）は spec 0022 で定義する**（例 `fn map<T, U>(xs, f: (T) -> U uses 'e) -> Array<U> uses 'e`．row 変数は sigil `'` 付きで `<>` の外）．error 型に対する行多相（union error）は本仕様では扱わない．

なお，`throws E` の `E` に通常の型パラメータを用いること（単一の error 型を総称する）は本仕様の範囲であり，有効である（`throws` は単一の型を取る，0011）．

```emela
fn map<T, U>(xs: Array<T>, f: (T) -> U uses {}) -> Array<U> uses {}
```

これは effect を具体的に `uses {}` へ固定した版で，effect-row 変数を使わない従来形である．effect-row 変数を用いた総称版は 0022 を参照．

#### 制限

- ジェネリックなのは **名前付き `fn` 定義** のみである．無名関数（lambda）は型パラメータを宣言できない．
- 型パラメータに境界 (bounds) は付かない．trait / impl が存在しないため，型パラメータは完全に不透明である．
- ジェネリック関数は **直接呼び出し**（`name(args)`）でのみ使える．第一級の値として束縛・受け渡しすること（例 `let f = identity`）は，型引数が決まらないため本仕様では許可しない．

### Examples

恒等関数:

```emela
fn identity<T>(x: T) -> T uses {} {
    x
}

fn main() -> Int uses {} {
    identity(42)        -- T = Int
}
```

配列の `map`:

```emela
fn map<T, U>(xs: Array<T>, f: (T) -> U uses {}) -> Array<U> uses {} {
    ...
}

fn main() -> Array<Int> uses {} {
    map([1, 2, 3], fn (x: Int) -> Int uses {} { x + 1 })   -- T = Int, U = Int
}
```

先頭要素:

```emela
fn first<T>(xs: Array<T>) -> Option<T> uses {} {
    Array.get(xs, 0)
}
```

### Compilation Notes

この節は非規範的な実装上の補足である．

- **単相化 (monomorphization)**: コンパイラはジェネリック関数を直接コードへ出力せず，呼び出し位置で定まった `(関数, 具体型引数)` の組ごとに，型変数を具体型へ置換した **単相版** を生成する．到達可能な特殊化は推移的に展開する（特殊化の本体が別のジェネリック関数を呼べば，その特殊化も生成する）．
- **typed IR (0012)**: 単相化は型検査と lowering の段階で行い，typed IR には型変数が一切現れない．これにより IR は「すべてのノードが具体型を持つ」という 0012 の条件を保つ．バックエンドは型変数を扱う必要がない．
- **WASM / JS backend**: いずれも具体型の単相版だけを受け取るため，ジェネリクス固有の処理は不要である．

### Open Questions

- 明示的な型引数構文（`identity<Int>(x)` 相当）を導入するか．比較演算子 `<` との構文的曖昧性をどう解決するか（turbofish 的構文など）．
- error 型に対する行多相（union error / `throws` の行変数）の導入．effect-row 多相 (`uses 'e`) は spec 0022 で定義済み．
- いずれのパラメータ型にも現れない型パラメータ（`pick<T>() -> T` のような戻り値専用の型パラメータ）を，明示的型引数の導入とあわせて許可するか．
- ジェネリックなデータ型宣言（ユーザー定義のジェネリック enum / record，0005 の `enum Either<L, R>`）．→ spec 0028 (Generic Data Type Declarations) で実現する．
- ジェネリック関数を第一級の値として扱う手段．
- 型パラメータの境界 (bounds) / trait．→ spec 0020 (Traits) で実現する．
