# 0035: Canonical Formatting and Lint

Status: Draft

Emela ソースコードの**唯一の正規スタイル**と，それを強制するツール `emela fmt`・静的検査
`emela lint` を定義する仕様．Go に倣い，スタイルは言語が決める：フォーマッタに設定項目は
存在せず（インデント幅・行幅・その他一切），すべての Emela コードは同じ形に整形される．
複数行構文と末尾カンマの文法は 0034 を前提とする．

## Summary

- `emela fmt [PATH ...]` はファイルを**その場で書き換え**，正規形へ整形する．PATH 省略時は
  カレントディレクトリを再帰する．`--check` は書き換えずに要整形ファイルを列挙し，非ゼロで
  終了する（CI 用）．
- 正規スタイル：インデント **4 スペース**，最大行幅 **100 桁**，幅超過時はブラケットグループを
  要素ごと 1 行に展開（末尾カンマ付き）．
- フォーマッタは意味を変えない：整形前後のプログラムは構文的に同一である（フォーマッタは
  検証まで行う）．コメントは保持される．アイテムの並べ替えは行わない．
- `emela lint [--package DIR ...] FILE ...` は型検査を通ったプログラムに対して警告規則を検査
  する．規則は固定であり，設定・無効化ファイルは存在しない．
- 診断に **warning** 深刻度と規則 ID（例 `naming/snake-case`）を導入する．

## Motivation

スタイル論争は言語が引き受けて終わらせる（Go / gofmt の立場）．設定可能なフォーマッタは
「プロジェクトごとの方言」を再生産し，コードを読む・移動する・コピーするたびに摩擦を生む．
Emela は仕様でスタイルを 1 つに固定し，ツールはそれを実行するだけとする．正規形が 1 つで
あることは，diff の安定・レビューの省力化・機械生成コードとの一致という実利も持つ．

lint は形式でなく意味の慣習を扱う：命名規則（識別子は `snake_case`，型とバリアントは
`PascalCase`），未使用の import / 束縛，宣言した effect の過剰（0023 が言及する lint）．
これらはコンパイルエラーにするほど硬くないが，放置すればコードベースの一貫性を損なう．

## Specification

### フォーマッタ（F 規則）

- **F1**: 正規スタイルは本仕様が定める唯一のものである (MUST)．`emela fmt` に整形結果を変える
  オプション・設定ファイルを設けてはならない (MUST NOT)．
- **F2**: インデントは 4 スペースである (MUST)．タブはインデントに使用しない．ネスト 1 段に
  つき 1 レベル．継続行（折り返されたグループの要素行）も +1 レベルである．
- **F3**: 最大行幅は **100 桁**である（UTF-8 の文字数．MUST）．整形後の行が 100 桁を超える
  場合，行内の最も外側のブラケットグループ（`(...)` / `[...]` / `uses { ... }`）を次の形へ
  展開する (MUST)：

  ```emela
  f(
      element,
      element,
  )
  ```

  開き記号の直後で改行し，要素ごとに 1 行（+1 インデント，末尾カンマ付き），閉じ記号は構文の
  基準インデントに置く．内側でなお超過する要素には同規則を再帰適用する．逆に，複数行で書かれた
  グループが 100 桁に収まるなら 1 行に結合する (MUST)．展開してもなお超える行（分割点を持た
  ない長い名前・文字列等）はそのまま許容される．
- **F4**: `{ ... }` ブロックの文構造（文の数と改行位置）は作者のものであり，フォーマッタは
  文を結合・分割しない (MUST)．ただし単一式ブロック `{ expr }` は 1 行形を保ってよい．
- **F5**: トークン間の空白は次表に従う (MUST)：

  | 位置 | 形 |
  |---|---|
  | 二項演算子（`+ - * / % ++ == != < > <= >= && \|\|`）の両側 | 1 スペース |
  | `=`（let）・`->` の両側 | 1 スペース |
  | ジェネリクスの `<` `>`（型パラメータ・型引数） | 密着（`List<Int>`） |
  | `,` `:` | 前は密着，後ろに 1 スペース |
  | `.` `::` `?` 前置 `!` | 密着 |
  | `(` `)` `[` `]` の内側 | 密着 |
  | 呼び出し・宣言の名前と `(` | 密着（`f(x)`，`fn add(`） |
  | ラムダの `fn` と `(` | 1 スペース（`fn (x: Int) -> Int`） |
  | `{` の前 | 1 スペース |
  | 1 行ブレースの内側 | 1 スペース（`uses { Stdout }`，`{ x * 2 }`）．空は密着（`uses {}`，`{}`） |
  | キーワード（`if` `else` `match` `try` `catch` `throw` `throws` `uses` `impl` `for` `enum` `trait` `pub` `let` `fn` `module` `import`）の後 | 1 スペース |

  比較の `<` `>` とジェネリクスの `<` `>` は構文解析の結果で区別される．
- **F6**: 空行の規則 (MUST)：連続する空行は 1 行に折り畳む．ブロック・アイテム本体の先頭と
  末尾の空行は除去する．ファイル先頭の空行は除去する．ファイル末尾は改行ちょうど 1 個で
  終わる．アイテム間の単一空行は保持する（強制はしない）．
- **F7**: コメント（`--`）は保持される (MUST)．行頭コメントはその位置のインデントに置き，
  直後のコードとの隣接（空行の有無）を保持する．コード行末のコメントはコードから 1 スペース
  空けて置く．コメント本文は行末空白の除去を除き不変である．コメントを含むグループは 1 行に
  結合しない．
- **F8**: enum バリアント・match / catch アームが改行で区切られている場合，冗長な区切り
  カンマは除去する (MUST)．改行区切りが正規形である（0005）．
- **F9**: アイテム（import・fn・enum・trait・impl・extern）の並べ替えは行わない (MUST NOT)．
- **F10**: 整形は冪等である (MUST)：`fmt(fmt(s)) = fmt(s)`．また整形前後のプログラムは構文的に
  同一でなければならず (MUST)，実装は書き込み前にこれを検証する (SHOULD)．
- **F11**: 出力は UTF-8・LF 改行であり，すべての行末空白を除去する (MUST)．

### フォーマッタ CLI

- **C1**: `emela fmt [PATH ...]`．PATH はファイルまたはディレクトリ（再帰的に `*.emel` を対象
  とし，`.` で始まるディレクトリは無視する）．省略時は `.`．整形結果が原文と異なるファイル
  のみ書き換え，そのパスを stdout に 1 行ずつ出力する．終了コード 0．
- **C2**: `emela fmt --check [PATH ...]`．書き換えず，要整形ファイルのパスを stdout に列挙する．
  1 つでもあれば終了コード 1，なければ 0．
- **C3**: 字句・構文エラーのあるファイルは整形せず，診断を stderr に出力して残りを継続し，
  全体の終了コードは 1 とする (MUST)．整形は構文解析のみを要し，import 解決・型検査を要しない
  (MUST)：型エラーのあるファイルも整形できる．

### リンタ（L 規則）

- **L1**: `emela lint [--package DIR ...] FILE ...` は各 FILE を通常のフロントエンド
  （構文解析 → import 解決 → 型検査，`main` は要求しない）にかけ，成功したプログラムに対して
  下記の規則を検査する (MUST)．構文・型エラーがある場合はそのエラーを報告して終了する．
- **L2**: 検査対象は指定したファイル自身の宣言に限る (MUST)．import で取り込まれたモジュール
  や prelude の違反は報告しない．
- **L3**: 規則は固定である．設定・抑制ファイルを設けてはならない (MUST NOT)．
- **L4**: 各警告は規則 ID を持ち，`warning:` 深刻度で報告される (MUST)．警告は 1 件で停止せず
  すべて収集され，ソース位置順に出力される (MUST)．
- **L5**: 終了コードは，警告 0 件なら 0，1 件以上なら 1 である (MUST)．

規則一覧：

| 規則 ID | 内容 |
|---|---|
| `naming/snake-case` | 関数・仮引数・`let` 束縛・extern の名前は `snake_case` でなければならない |
| `naming/pascal-case` | enum・バリアント・trait の名前は `PascalCase` でなければならない |
| `imports/unused` | import した名前がファイル内で一度も参照されていない |
| `bindings/unused-let` | `let` 束縛が以後一度も使用されていない（`_` 始まりは免除） |
| `bindings/unused-param` | 仮引数が本体で使用されていない（`_` 始まりは免除．trait メソッドの実装・デフォルト本体は署名が trait に固定されるため免除） |
| `effects/over-declared` | `uses` に宣言した effect のうち本体が要求しないものがある（0023） |

### 診断の深刻度

- **D1**: 診断は `error` と `warning` の 2 深刻度を持つ (MUST)．lint の規則違反は `warning`，
  それ以外の既存診断は `error` である．
- **D2**: warning の表示形式は既存の error と同形で，先頭行を
  `warning: <title> [<rule-id>]` とする (MUST)．

## Examples

```emela
-- 整形前
fn add (x:Int,y:Int)->Int uses {} {x+y}

-- 整形後
fn add(x: Int, y: Int) -> Int uses {} { x + y }
```

```emela
-- 100 桁を超えるシグネチャは要素ごとに折り返される（0034）
fn transfer_between_accounts(
    source_account: AccountId,
    destination_account: AccountId,
    requested_amount: Int,
) -> TransferReceipt throws TransferError uses { Ledger, Clock } {
    ...
}
```

```
$ emela lint main.emel
warning: Unused import [imports/unused]

  --> main.emel:2:1
  2 | import std.string.repeat
    | ^^^^^^^^^^^^^^^^^^^^^^^^ `repeat` is imported but never used

error: 1 warning emitted
```

## Open Questions

- import の正規順序（アルファベット順ソート）を F 規則に加えるか．並べ替えは F9 の例外に
  なるため，導入するなら明示の規則として定める．
- エディタ統合のための stdin / stdout 整形モード（`emela fmt -`）．
- 警告の行単位抑制コメント（`-- lint:allow(rule-id)`）．L3 の「設定を持たない」と両立する
  最小の逃げ道として検討する．
- 型パラメータ名（`<T>` の `T`）の命名検査．現行 AST が span を持たないため v1 では宣言名の
  位置に警告を anchoring する．
